= Catalog

The catalog is the audit trail. It connects every architectural artifact to every other artifact -- diagrams to ADRs, ADRs to design documents, design documents to code, code to diagrams. If something is missing, the catalog shows it. If something contradicts, the catalog reveals it.

== Known Inconsistencies

Open audit findings tracked in `INDEX.md`. Summary:

* *10 documentation mismatches* (D-Audit-1 through 10) -- port names, event counts, routing, actor count
* *9 code issues* (C-Audit-1 through 9) -- dead code, duplicate events, wrong packages, stringly-typed fields
* *3 test gaps* (T-Audit-1 through 3) -- smoke-only server test, self-referential assertion, low coverage

These will be resolved through the detailed design pass (steps A through D6).
