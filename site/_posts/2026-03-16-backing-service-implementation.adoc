== The Scope

The full eventstorming board has four actors across the ACL:

1. **Hippocampus** -- the domain customer. Out of scope.
2. **Memory-Data Adapter** -- the inner adapter (the cook). Out of scope.
3. **SQLite Driver** -- the outer adapter (the waitress). **In scope.**
4. **SQLite** -- the infrastructure (the filing clerk). **In scope.**

The port is the boundary. We implement from the port outward.

== Reading the Board as Tests

Each GWT chain is already a test specification. The ceremony did the work.

=== SQLite Driver -- Policy 1: Persist

**Given** a DTO arrives at the driver +
**When** the driver translates it to a prepared statement +
**Then** persistence is requested, carrying the transaction ID

=== SQLite -- Execute

**Given** a prepared statement arrives +
**When** SQLite executes it +
**Then** either data is persisted (with SQL record) or persistence fails (with SQL error)

=== SQLite Driver -- Policy 2: Receive

**Given** SQLite responds (record or error) +
**When** the driver receives and translates back to DTO +
**Then** either a result DTO or an error DTO is returned, linked by transaction ID
