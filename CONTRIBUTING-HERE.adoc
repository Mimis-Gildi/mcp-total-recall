= Total Recall -- Project-Specific Contributions
:description: Project-specific contributing guidelines for Total Recall.
:!toc:
:keywords: contributing

This is a synthetic memory system. Its architecture carries lessons from three generations of development (2015--present).

== Contributor License Agreement

All contributions require acceptance of the link:CLA.md[Contributor License Agreement]. By opening a pull request, you agree to the CLA terms. This allows the project to maintain licensing flexibility while keeping the source open.

== Architecture Rules

These are non-negotiable. They exist because previous generations failed without them.

* All service layers are decoupled.
* Backing services are coded against an interface, never against a specific implementation.
* Transport is `stdio` primary. No SSE.
* Design for graceful shutdown in every component.

See link:README.md[README] for the full architectural history.

== Testing

We use link:https://kotest.io[Kotest 6] with multiple spec styles, each matched to the domain concept it tests.
The test suite itself is the convention -- read the existing tests to understand the patterns.

* Factory helpers in `mimis.gildi.memory.testing.Fixtures` -- `aMemory()`, `aTransactionContext()`, `anAssociation()`, `aSalienceScore()`
* Project-wide config in `mimis.gildi.memory.testing.ProjectConfig`
* Run tests: `./gradlew test`
* Report: `build/reports/tests/test/index.html`

See the link:/mcp-total-recall/engineering/2026/03/09/test-conventions/[Test Conventions] blog post for the thinking behind our approach.
